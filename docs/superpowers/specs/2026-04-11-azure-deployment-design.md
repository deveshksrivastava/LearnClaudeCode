# Azure Deployment Design — ShopFast

**Date:** 2026-04-11
**Goal:** Deploy ShopFast (e-commerce API + LLM chatbot + React frontend) to Azure for a learning/portfolio project
**Author:** Devesh Srivastava

---

## 1. Goals & Constraints

- **Purpose:** Learning/portfolio — showcase Azure deployment skills at minimal cost
- **Target cost:** ~$5–10/month
- **Data persistence:** In-memory only (SQLite + ChromaDB reset on container restart) — acceptable for demos
- **CI/CD:** GitHub Actions — automatic deploy on push to `main`

---

## 2. Azure Services Used

| Resource | Tier | Est. Cost/month |
|----------|------|-----------------|
| Azure Static Web Apps | Free | $0 |
| Container Apps Environment | Consumption | $0 (idle) |
| ecommerce-api Container App | Consumption | ~$0–2 |
| llm-chatbot Container App | Consumption | ~$0–3 |
| Azure Container Registry | Basic | ~$5 |
| Azure Key Vault | Standard | ~$0 (<10k ops) |
| **Total** | | **~$5–10/month** |

All resources live in a single resource group: `shopfast-rg` in `eastus`.

## 3. High-Level Architecture

```

└── Resource Group: shopfast-rg
    ├── Azure Static Web Apps (FREE)
    │   └── React/Vite SPA — global CDN, auto HTTPS
    │
    ├── Azure Container Registry (Basic)
    │   ├── shopfast/ecommerce:latest
    │   └── shopfast/chatbot:latest
    │
    ├── Container Apps Environment: shopfast-env (Consumption)
    │   ├── Container App: ecommerce-api
    │   │   └── FastAPI + SQLite (in-memory) — port 8000
    │   │   └── min-replicas: 0, max-replicas: 3
    │   └── Container App: llm-chatbot
    │       └── FastAPI + LangChain + LangGraph + ChromaDB (in-memory)
    │       └── min-replicas: 0, max-replicas: 2
    │       └── Calls ecommerce-api via internal DNS
    │
    ├── Azure Key Vault: shopfast-kv
    │   ├── AZURE-OPENAI-KEY
    │   └── AZURE-OPENAI-ENDPOINT
    │
    └── Azure OpenAI Service (pre-existing)
        ├── gpt-4o deployment
        └── text-embedding-ada-002 deployment
```

## 4. Data Flow

1. **User browses products:** Browser → Static Web Apps (CDN) → ecommerce-api Container App
2. **User chats with bot:** Browser → Static Web Apps → llm-chatbot Container App → Azure OpenAI
3. **Chatbot queries products:** llm-chatbot → ecommerce-api via internal Container Apps DNS (`http://ecommerce-api`) — no public internet hop
4. **RAG retrieval:** llm-chatbot → ChromaDB (in-memory, same container process)
5. **Secrets at startup:** Container Apps pull secrets from Key Vault via Managed Identity — no credentials in code or env files

## 5. CI/CD Pipeline (GitHub Actions)

### Trigger
`git push` to `main` branch

### Jobs

**Job 1: test** (all in parallel)
- `pytest` — e-commerce API tests
- `pytest llm-chatbot-backend/` — chatbot tests
- `npm run build` — React/Vite production build

**Job 2: build-and-push** (runs after test passes)
- `az acr login` to Azure Container Registry
- `docker build` → `shopfast/ecommerce:${{ github.sha }}`
- `docker build` → `shopfast/chatbot:${{ github.sha }}`
- `docker push` both images to ACR

**Job 3: deploy** (runs after build-and-push)
- `az containerapp update` → ecommerce-api (new image tag)
- `az containerapp update` → llm-chatbot (new image tag)
- Static Web Apps deploy action → React build artifacts

### GitHub Secrets Required

| Secret | Description |
|--------|-------------|
| `AZURE_CREDENTIALS` | Service principal JSON from `az ad sp create-for-rbac` |
| `REGISTRY_LOGIN_SERVER` | `shopfastacr.azurecr.io` |
| `AZURE_OPENAI_KEY` | Azure OpenAI API key (also stored in Key Vault) |
| `AZURE_OPENAI_ENDPOINT` | Azure OpenAI endpoint URL (also stored in Key Vault) |
| `AZURE_STATIC_WEB_APPS_TOKEN` | Auto-generated by Static Web Apps on creation |

## 6. Infrastructure Provisioning (One-Time Setup)

Run these Azure CLI commands once to provision all resources:

```bash
# 1. Resource Group
az group create --name shopfast-rg --location eastus

# 2. Azure Container Registry
az acr create \
  --resource-group shopfast-rg \
  --name shopfastacr \
  --sku Basic

# 3. Key Vault + Secrets
az keyvault create \
  --resource-group shopfast-rg \
  --name shopfast-kv \
  --location eastus
az keyvault secret set --vault-name shopfast-kv --name AZURE-OPENAI-KEY --value "<key>"
az keyvault secret set --vault-name shopfast-kv --name AZURE-OPENAI-ENDPOINT --value "<endpoint>"

# 4. Container Apps Environment
az containerapp env create \
  --resource-group shopfast-rg \
  --name shopfast-env \
  --location eastus

# 5a. ecommerce-api Container App
az containerapp create \
  --resource-group shopfast-rg \
  --name ecommerce-api \
  --environment shopfast-env \
  --image shopfastacr.azurecr.io/shopfast/ecommerce:latest \
  --registry-server shopfastacr.azurecr.io \
  --min-replicas 0 --max-replicas 3 \
  --ingress external --target-port 8000 \
  --env-vars DB_PATH=":memory:" LOG_LEVEL=INFO

# 5b. llm-chatbot Container App
az containerapp create \
  --resource-group shopfast-rg \
  --name llm-chatbot \
  --environment shopfast-env \
  --image shopfastacr.azurecr.io/shopfast/chatbot:latest \
  --registry-server shopfastacr.azurecr.io \
  --min-replicas 0 --max-replicas 2 \
  --ingress external --target-port 8000 \
  --env-vars \
    LLM_PROVIDER=azure \
    AZURE_OPENAI_DEPLOYMENT_NAME=gpt-4o \
    AZURE_OPENAI_API_VERSION=2024-02-01 \
    AZURE_OPENAI_EMBEDDING_DEPLOYMENT=text-embedding-ada-002 \
    ECOMMERCE_API_URL=http://ecommerce-api \
    CHROMA_PERSIST_PATH=":memory:" \
    LOG_LEVEL=INFO

# 6. Azure Static Web Apps
az staticwebapp create \
  --resource-group shopfast-rg \
  --name shopfast-web \
  --source https://github.com/<your-username>/LearnClaudeCode \
  --branch main \
  --app-location /frontend \
  --output-location dist
```

## 8. CORS Configuration

Both FastAPI backends must allow the Static Web Apps domain:

```python
# app/main.py and llm-chatbot-backend/app/main.py
allow_origins = [
    "https://<name>.azurestaticapps.net",  # production
    "http://localhost:5173",               # local dev
]
```

---

## 9. Managed Identity & Security

- Each Container App gets a **system-assigned Managed Identity**
- Key Vault grants `get` permission to both identities via access policies
- No secrets stored in code, Dockerfiles, or plain environment variables
- ACR admin access disabled — Container Apps pull images via Managed Identity
- CORS restricted to Static Web Apps domain only in production

---

## 10. Rollback Strategy

- Every Docker image is tagged with the git SHA (`${{ github.sha }}`)
- To rollback: re-run the previous successful GitHub Actions workflow run
- This re-points Container Apps to the previous image tag automatically
